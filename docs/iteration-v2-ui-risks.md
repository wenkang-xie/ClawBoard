# Agent Board v2 交互风险清单（UI 视角）

> 重点：误操作防护 + 生产环境高风险动作。
> 适用范围：Fleet / Configs / Diff&Apply / Diagnostics&Logs / Operations。

---

## 0. 风险分级定义

- **R1（致命）**：可能导致生产中断、错误配置生效、错误对象被操作
- **R2（高）**：可能造成局部可用性下降、误触发高成本动作
- **R3（中）**：造成排障效率下降、信息误读
- **R4（低）**：体验问题，不直接影响系统安全

---

## 1. 风险总览（优先治理项）

- R1-01：多 Gateway 切换后上下文错判，导致在错误 Gateway 执行 Apply/Restart/Rollback
- R1-02：Prod 环境动作确认不足，误将草稿应用到生产
- R1-03：重启/回滚对象识别不清（Gateway 别名相似、Region 混淆）
- R2-01：Diff 信息过载，关键敏感字段变化被忽略
- R2-02：并发编辑冲突未被感知，覆盖他人更改
- R2-03：诊断与日志页跨页面跳转丢失上下文，错误归因

---

## 2. 详细风险条目（含防护方案）

## 2.1 多 Gateway 上下文误判（最高优先）

### 场景
- 用户在 A Gateway 编辑配置后切换到 B，但页面局部状态未刷新；随后误在 B 执行 A 的操作意图。

### 后果
- R1：错误目标执行高风险动作。

### 必做防护
- 顶部固定 Context Chip：`Gateway + ENV + Region` 常驻可见
- Gateway 切换触发“全局上下文版本号”刷新，旧请求结果禁止写入 UI
- 切换后清空所有危险操作态（按钮 disabled、Modal 失效）
- 未保存草稿必须阻断切换（保存草稿/放弃/取消）

### 验收
- 人为制造慢请求时，切换后不会出现旧 Gateway 数据回填
- 切换后若 Confirm Modal 已打开，必须自动关闭并提示重新确认

---

## 2.2 生产环境误操作（Apply）

### 场景
- 在 Prod 环境点击 Apply，因确认流程过弱导致误发布。

### 后果
- R1：配置错误直达生产，触发故障。

### 必做防护
- 高风险动作统一双确认：
  - Step1：环境与目标确认（高对比 ENV 标签）
  - Step2：文本确认 `APPLY <gateway-alias>`（精确匹配）
- Prod 额外门槛：
  - 勾选“已确认影响范围”
  - 变更原因必填
  - 可配置工单号必填
- Apply 前必须展示风险摘要（影响服务数、需重启与否、敏感字段数）

### 验收
- 任一确认条件不满足，Apply 按钮不可用
- 误输入文本（大小写/空格/别名错误）均不可提交

---

## 2.3 重启误触发（Restart）

### 场景
- 用户选择了错误重启模式（强制重启）或错误对象执行重启。

### 后果
- R1/R2：中断扩大，恢复时间拉长。

### 必做防护
- 重启模式默认“优雅重启”，强制重启折叠并增加危险提示
- 执行前展示影响预估（中断时长、影响会话）
- 双确认文本：`RESTART <gateway-alias>`
- 若健康检查存在阻塞项，禁止执行并给出处理路径

### 验收
- 强制重启必须经过额外显式展开与确认
- 健康阻塞项存在时无法点击最终执行

---

## 2.4 错误版本回滚（Rollback）

### 场景
- 在版本时间线中选错版本，回滚到非目标点。

### 后果
- R1：配置状态回退错误，可能引发连锁故障。

### 必做防护
- 回滚前必须展示反向 Diff 与“将撤销字段清单”
- 双确认文本：`ROLLBACK <version-id>`
- 版本项显示完整上下文：版本号、操作人、时间、变更摘要
- 回滚后动作（是否自动重启）必须显式选择，不允许隐式执行

### 验收
- 未查看反向 Diff 不允许进入最终确认
- 文本确认与版本 ID 不一致时禁止提交

---

## 2.5 Diff 关键信息漏读

### 场景
- 变更量大导致敏感字段变化（auth/token/路由策略）被忽略。

### 后果
- R2：高风险变更未被识别。

### 必做防护
- 敏感字段高风险标记 + 过滤器（仅看敏感项）
- 风险摘要固定置顶
- “仅看变更项”与“按风险排序”开关

### 验收
- 敏感字段命中率 100%（规则字典回归测试）

---

## 2.6 并发编辑覆盖

### 场景
- 两人同时编辑同一模块，后提交者覆盖先提交者。

### 后果
- R2：配置丢失或不一致。

### 必做防护
- 保存/应用前做版本比对（etag/version）
- 检测冲突时阻断提交，要求先 refresh + merge
- 冲突 UI 展示“你修改了什么 vs 远端改了什么”

### 验收
- 并发冲突时不会 silent overwrite

---

## 2.7 诊断/日志链路断裂导致误归因

### 场景
- 从诊断跳转日志时丢失 Gateway 或时间窗口，导致看到错误数据。

### 后果
- R2/R3：排障方向错误、耗时增加。

### 必做防护
- 查询条件 URL 参数化并包含上下文
- 页面顶部显示“当前查询绑定上下文”
- 提供一键恢复“诊断发起时条件”

### 验收
- 跳转后 gatewayId/env/timeRange 完整继承

---

## 3. 风险控制清单（上线前 Gate）

- [ ] 所有高风险动作都接入统一 Confirm Modal（双确认 + 文本）
- [ ] Prod 环境附加确认项已开启
- [ ] Gateway 切换的上下文版本戳保护已启用
- [ ] 旧请求回填保护（取消请求或响应丢弃）已验证
- [ ] 危险按钮在上下文变更后自动 disabled
- [ ] Apply/Restart/Rollback 全链路审计字段齐全（动作/对象/原因/操作者/时间）
- [ ] E2E 已覆盖至少 4 个误操作关键场景

---

## 4. 建议测试场景（误操作专项）

1. **跨 Gateway 误触发防护**
   - 在 Gateway A 打开 Apply 确认框 -> 切换到 Gateway B -> 验证确认框失效且动作不可提交。

2. **Prod 文本确认鲁棒性**
   - 输入近似文本（大小写错误、别名缺字母）-> 验证始终不能提交。

3. **慢网与竞态回填**
   - 人工注入延迟：A 请求慢、B 请求快；切换到 B 后 A 返回 -> 验证 UI 不被 A 覆盖。

4. **重启失败恢复闭环**
   - 模拟 Restart 失败 -> 应出现回滚入口、诊断入口、重试入口三选项。

5. **回滚错版本防护**
   - 未查看反向 Diff 时尝试提交 -> 应被阻断。
